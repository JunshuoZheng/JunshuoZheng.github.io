常用远程开发工具汇总

2023-02-26

在小电脑（Jetson）上进行 Linux 开发时，需要连接一堆外设，很不方便。我们可以使用自己的电脑进行远程开发，这样 Jetson 插上电源开机就行，不用连显示器和外设。

以下所有工具和方法均需要固定 IP 地址。

⭐Xshell（最常用）

远程连接 Linux 终端

https://www.xshell.com/zh/xshell/

现在可以下载免费的 Xshell 了，只要注册一下就行：

https://www.xshell.com/zh/free-for-home-school/

⭐Xming

下载地址：Xming X Server for Windows

调试视觉和 ROS 的时候经常要用到图形化的工具，而使用 VNC Viewer 的话在开发的流程中太割裂。

Xming是一个X11转发工具，可以把 Linux 的X11窗口转发到 Windows 上，实现在 Windows PC 下使用 Linux 图形化程序。配合 Xshell 使用，效果如下：

开启之后会造成 Xshell 轻微卡顿

使用方法

  1. 下载安装Xming，打开 XLaunch 一路 Next 无需修改，启动 Xming

  1. 在Xming的安装目录找到X0.hosts文件（X0.hosts中的0就是刚才需要记住的Display number），并在localhost下一行添加你的服务器IP（即 Jetson 的IP）

  2. 在 Xshell 设置 X11 转发：勾选 X DISPLAY，填入localhost``:x.0，（x=第一步中的display nuber，默认为0）

  3. 远程配置：打开远程终端，输入

其中，127.0.0.1为本机地址，10.0是默认的。

在终端打开 xauth list 可以看到，在我的 Xavier NX 上可以看到这几个号，把10换成11或者12也能用。

这里遇到的坑：

  1. 网上的教程大多解释说是和第一步的 display number 对应的，其实和那个没关系。（至少在我的环境中，这二者毫无关系）

  2. 127.0.0.1有的教程写的localhost ，但是我的环境中没有指定localhost。

  3. 远程配置：打开远程终端，打开vi /etc/ssh/sshd_config，修改X11Forwarding 改为为“yes”，如被#注释，需取消注释。

  4. 完成。

Jetbrains Gateway（不可用）

深入了解 JetBrains Gateway | Company Blog

目前（2023年2月）完全不可用。暂时不推荐使用，不如 VSCode！！

可以在 Jetson 上运行 Jetbrains 的 IDE，然后连接到PC上的瘦客户端。概念类似于云电脑，本地只负责编写代码，所有的存储、编译过程全在 Jetson 上。

优点：

  1. 配置比较简单，且不需要配置本地环境。

  2. 方便监控Jetson的状态

  3. 不依赖本地环境，用别人的电脑也可以很快进入到开发状态

  4. 支持 Jetbrains 的 Clion 、Pycharm 等多个 IDE，JB 家的 IDE 做的很棒，比 VSCode 好用多了

  5. 集成在 Jetbrains 的 IDE 之中，如果你有 Clion 或者 Pycharm，无需安装额外的软件

缺点：

  1. 致命缺点1：延迟太大了，写一句话都有明显延迟

  2. 致命缺点2：占用 CPU 资源太多。在 Jetson Xavier NX 什么都没运行时 CPU 占用大概在 30% 左右，打开 Clion 之后大概在 55% 左右，打开一个文件时，CPU 占用会飙到100%。

  3. 打开速度慢，在 NX 上打开竟然要用三分钟…

  4. 和 VSCode Remote 的原理不一样，Gateway 需要传输画面
